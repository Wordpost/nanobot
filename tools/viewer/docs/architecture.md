# Architecure Decision Record (ADR): Vivir (Forensic Viewer)

## Статус
Принято (Accepted)

## Контекст
Vivir (Nanobot Forensic Viewer) выступает в роли единого интерфейса для динамического инспектирования логов агентов, субагентов и их подсистем памяти.

### 1. Миграция фронтенда (Preact & Vite)
- Весь слой пользовательского интерфейса (UI) был переписан с использованием Preact и Vite; исходный код находится в папке `/frontend`.
- Результат сборки (`static_dist`) напрямую отдается (расшаривается) приложением FastAPI.

### 2. Поддержка множественных пространств ("Pool Mode")
- Установка переменной окружения `NANOBOT_POOL_DIR` заставляет файл `config.py` автоматически находить все дочерние папки (рабочие пространства различных агентов).
- Скрипт сканирует структуру папок в поисках `child_dir/workspace/sessions` и собирает их в словарь объектов `WorkspaceInfo`, назначая каждому агенту уникальное имя (напр. контейнера или процесса).
- Архитектурный маппинг выглядит так: папка `.nanobot` → имя `nanobot-gateway`, папка `.shantra` → имя `nanobot-shantra`.

### 3. Сеть и безопасность (Loopback)
- Для строгого соответствия требованиям безопасности на удаленных VPS, `NANOBOT_HOST` по умолчанию равен `127.0.0.1`.
- Разработчики и операторы, подключающиеся со своих локальных машин, используют **SSH-туннели с портом проброса**: `ssh -L [PORT]:127.0.0.1:[PORT] user@ip`.
- Это обеспечивает нулевую поверхность атаки извне (через HTTP) и избавляет от необходимости усложнять слой L7 HTTP механизмом аутентификации.
